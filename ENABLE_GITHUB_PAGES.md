# 🚀 Enable GitHub Pages - Access Your Course as a Website

## ⚡ Quick Setup (2 Minutes)

### Step 1: Go to Repository Settings
```
1. Visit: https://github.com/rupam13/ServiceNow-JavaScript-
2. Click "Settings" tab (top right)
3. Look for "Pages" in left sidebar
```

### Step 2: Enable GitHub Pages
```
1. Click "Pages" in left menu
2. Under "Source", select:
   - Branch: main
   - Folder: / (root)
3. Click "Save"
4. Wait 1-2 minutes for deployment
```

### Step 3: Access Your Course Website
```
Your site will be live at:
https://rupam13.github.io/ServiceNow-JavaScript-/
```

## ✅ Troubleshooting

### Issue: Still Getting 404

**Solution 1: Wait for Deployment**
```
1. Go to Settings > Pages
2. Wait for green checkmark
3. Refresh browser (Ctrl+Shift+R)
4. Try again in 2-3 minutes
```

**Solution 2: Check Workflow**
```
1. Go to "Actions" tab in GitHub
2. Look for "Deploy to GitHub Pages" workflow
3. Should show green checkmark
4. If red X: click to see error
```

**Solution 3: Force Redeployment**
```
1. Go to Settings > Pages
2. Change branch to "main"
3. Click Save
4. Wait for redeployment
```

## 🔄 How It Works

### Automatic Deployment
```
You push code
    ↓
GitHub Actions triggered
    ↓
Workflow runs (deploy.yml)
    ↓
Files copied to _site/
    ↓
Deployed to GitHub Pages
    ↓
Website live at: https://rupam13.github.io/...
```

### Every Time You Push
- New commits to `main` automatically trigger deployment
- Changes appear on website within 1-2 minutes
- No manual steps needed
