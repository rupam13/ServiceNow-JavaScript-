# ServiceNow JavaScript Daily Practice Program

## Goal
Master ServiceNow JavaScript from basic to advanced through daily practice and structured learning.

## Repository Structure

```
servicenow-js-practice/
├── 00_setup/              # Setup instructions and environment
├── 01_basics/             # Fundamentals (Week 1-2)
├── 02_intermediate/       # Core concepts (Week 3-5)
├── 03_advanced/           # Advanced topics (Week 6-8)
├── 04_reference/          # Quick reference guides
└── 05_practice-logs/      # Daily progress tracking
```

## Learning Path Overview

### Phase 1: Basics (Days 1-10)
- Syntax & Statements
- Variables & Data Types
- Operators (Arithmetic, Comparison, Logical)
- Strings & Special Characters
- Type Conversions

### Phase 2: Intermediate (Days 11-25)
- Control Flow (If/Else, Switch)
- Loops (For, While, Do-While)
- Functions & Scope
- Arrays & Array Methods
- Objects & JSON

### Phase 3: Advanced (Days 26-40)
- Classes & Prototypes
- Inheritance
- Error Handling (Try/Catch)
- Database Queries (GlideRecord)
- REST APIs

## Daily Practice Routine

1. **Review** (5 min): Read the lesson concept
2. **Understand** (10 min): Study examples and explanations
3. **Practice** (20 min): Write code for the practice task
4. **Implement** (15 min): Create a ServiceNow-specific use case
5. **Log** (5 min): Document your progress

## How to Use

1. Start with `01_basics/day_01_getting_started.js`
2. Follow the numbered sequence
3. Complete the practice exercises
4. Log your progress in `05_practice-logs/progress.md`
5. Reference quick guides in `04_reference/` as needed

## Resources

- Course Material: Based on "Learn JavaScript on the Now Platform" YouTube series
- PDI: Download from https://developer.servicenow.com
- Script Location: System Definition > Scripts - Background

## Tips for Success

✅ **Do This:**
- Write code every day, even if just 15 minutes
- Test code in a ServiceNow instance
- Review previous lessons weekly
- Create your own variations of examples

❌ **Avoid This:**
- Skipping days
- Copy-pasting without understanding
- Moving forward without practice
- Ignoring error messages
